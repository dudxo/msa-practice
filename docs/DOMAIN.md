# DOMAIN.md

## 1. 서비스 한 줄 소개

- MSA 학습을 위한 게시판 서비스로, 회원/게시글/포인트를 독립 마이크로서비스로 분리하여 운영합니다.
- 주요 목적: MSA 핵심 패턴(서비스 간 통신, Saga, 이벤트 기반 아키텍처, API Gateway) 학습 및 포트폴리오

---

## 2. 주요 도메인(바운디드 컨텍스트)

- **User**: 회원가입, 로그인, 사용자 조회
- **Board**: 게시글 CRUD, 게시글 조회 최적화
- **Point**: 포인트 적립/차감, 활동 점수

---

## 3. 핵심 도메인 용어 정의

| 용어 | 설명 |
|-----|------|
| User Service | 회원 관리 담당 마이크로서비스 (포트 8081) |
| Board Service | 게시글 관리 담당 마이크로서비스 (포트 8082) |
| Point Service | 포인트/활동점수 관리 마이크로서비스 (포트 8083, 추후) |
| 활동 점수 | 게시글 작성 등 활동 시 적립되는 점수 |
| 보상 트랜잭션 | Saga 패턴에서 실패 시 이전 작업을 되돌리는 트랜잭션 |

---

## 4. 전체 아키텍처 개요

- 클라이언트: REST API (curl/Postman)
- 서버: Java 17 / Spring Boot 3.5.x 기반 마이크로서비스
- 데이터베이스: MySQL 8.0 (서비스별 독립 DB)
- 인프라: Docker Compose

```
Client → [API Gateway] → User Service  → User DB (3321)
                       → Board Service → Board DB (3322)
                       → Point Service → Point DB (추후)
```

---

## 5. 전역적인 비즈니스 규칙

- 각 서비스는 자기 DB만 접근한다 (다른 서비스 DB 직접 접근 금지)
- 서비스 간 통신은 REST API 또는 Kafka 이벤트를 통해서만 수행한다
- 게시글 작성 시 포인트가 차감되며, 포인트 부족 시 작성 불가 (추후)
